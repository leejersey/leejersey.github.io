# 第一章 环境搭建与基础连接

> 本章目标：把 Redis 跑起来，用 Python 连上它，理解同步/异步两种连接方式和连接池的工程意义。

---

## 1.1 安装 Redis 与 redis-py

### 安装 Redis Server

**macOS（Homebrew）：**

```bash
brew install redis
brew services start redis    # 后台启动，开机自启
```

**Ubuntu/Debian：**

```bash
sudo apt update
sudo apt install redis-server
sudo systemctl enable redis-server
sudo systemctl start redis-server
```

**Docker（推荐，跨平台一致）：**

```bash
docker run -d --name redis \
  -p 6379:6379 \
  redis:7-alpine
```

验证是否启动成功：

```bash
redis-cli ping
# 返回 PONG 表示成功
```

### 安装 Python 客户端

```bash
pip install redis
```

`redis-py` 是 Redis 官方推荐的 Python 客户端，从 v4.2+ 开始内置了异步支持（`redis.asyncio`），不再需要单独安装 `aioredis`。

> 💡 **版本说明**：本教程基于 `redis-py >= 5.0`，Python >= 3.10。如果你用的是旧版本，部分异步 API 可能略有差异。

## 1.2 同步连接与 decode_responses

### 最简连接

```python
import redis

# 创建连接
r = redis.Redis(host="127.0.0.1", port=6379, db=0)

# 写入
r.set("hello", "world")

# 读取
value = r.get("hello")
print(value)   # b'world'  ← 注意：返回的是 bytes！
print(type(value))  # <class 'bytes'>
```

### decode_responses：告别 bytes

默认情况下，redis-py 返回的值是 `bytes` 类型。每次都要 `.decode()` 很烦。加上 `decode_responses=True` 一劳永逸：

```python
import redis

r = redis.Redis(
    host="127.0.0.1",
    port=6379,
    db=0,
    decode_responses=True,  # ← 关键参数
)

r.set("hello", "world")
value = r.get("hello")
print(value)        # 'world'  ← 直接是 str
print(type(value))  # <class 'str'>
```

> ⚠️ **什么时候不用 `decode_responses=True`？** 当你需要存储二进制数据（图片、序列化对象等）时，保持默认的 `bytes` 模式更合适。

### 连接参数速查

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `host` | `"localhost"` | Redis 服务地址 |
| `port` | `6379` | Redis 端口 |
| `db` | `0` | 数据库编号（0-15） |
| `password` | `None` | 认证密码 |
| `decode_responses` | `False` | 是否自动解码为 str |
| `socket_timeout` | `None` | 读写超时（秒） |
| `socket_connect_timeout` | `None` | 连接超时（秒） |
| `retry_on_timeout` | `False` | 超时是否自动重试 |

### 用 URL 连接

更简洁的写法，尤其适合从环境变量读取：

```python
import redis

# 标准格式：redis://[:password@]host:port/db
r = redis.from_url("redis://127.0.0.1:6379/0", decode_responses=True)

# 带密码
r = redis.from_url("redis://:mypassword@127.0.0.1:6379/0", decode_responses=True)

# 生产环境推荐：从环境变量读取
import os
r = redis.from_url(os.getenv("REDIS_URL", "redis://127.0.0.1:6379/0"), decode_responses=True)
```

## 1.3 连接池：为什么需要、怎么配置

### 问题：每次操作都建新连接？

```python
# ❌ 反模式：每次请求都创建新连接
def handle_request():
    r = redis.Redis(host="127.0.0.1", port=6379)  # 每次都建 TCP 连接
    r.get("key")
    # 函数结束，连接被垃圾回收关闭
```

每次创建/销毁 TCP 连接需要三次握手 + 四次挥手，高并发下这个开销不可忽视。

### 解决方案：连接池

连接池预先创建一批 TCP 连接，需要时从池中取，用完放回，避免反复创建/销毁：

```
┌──────────────────────────────┐
│        ConnectionPool        │
│                              │
│   Conn 1  ← 正在使用         │
│   Conn 2  ← 空闲             │
│   Conn 3  ← 空闲             │
│   Conn 4  ← 正在使用         │
│   ...                        │
│   Conn N  ← 空闲             │
│                              │
│   max_connections = 50       │
└──────────────────────────────┘
```

### 显式创建连接池

```python
import redis

# 创建连接池
pool = redis.ConnectionPool(
    host="127.0.0.1",
    port=6379,
    db=0,
    max_connections=50,       # 最大连接数
    decode_responses=True,
)

# 所有 Redis 实例共享同一个池
r = redis.Redis(connection_pool=pool)

# 在整个应用中复用这个 r 实例
r.set("pool:test", "ok")
print(r.get("pool:test"))  # 'ok'
```

### 其实你已经在用连接池了

一个小秘密：当你直接写 `redis.Redis(...)` 时，redis-py 内部**自动创建了一个连接池**。只是默认的 `max_connections` 是 `2**31`（约 21 亿），相当于不限制。

显式创建连接池的意义在于：**控制最大连接数，防止打满 Redis 的 `maxclients` 配置**。

### 连接池大小怎么设？

| 场景 | 建议 max_connections |
|------|---------------------|
| 单线程脚本 | 5-10 |
| Web 应用（同步，如 Flask） | 20-50 |
| Web 应用（异步，如 FastAPI） | 10-30（异步复用更高效） |
| 后台任务（Celery Worker） | 10-20 |

> 经验法则：**不要超过 Redis 的 `maxclients`（默认 10000）的 10%**。多个应用实例要一起算。

## 1.4 异步连接：redis.asyncio 入门

### 为什么需要异步？

在 FastAPI 等异步框架中，如果用同步的 `redis.Redis`，每次 Redis 操作都会**阻塞整个事件循环**，导致其他请求无法被处理：

```
同步（阻塞）：
请求 A → Redis 操作（等 5ms）→ 响应 A → 请求 B → Redis 操作 → 响应 B
         ↑ 这 5ms 里其他请求都在排队

异步（非阻塞）：
请求 A → Redis 操作（等待期间处理请求 B）→ 响应 A
请求 B → Redis 操作（等待期间处理请求 C）→ 响应 B
```

### 基本用法

```python
import asyncio
import redis.asyncio as aioredis

async def main():
    # 创建异步连接
    r = aioredis.Redis(
        host="127.0.0.1",
        port=6379,
        db=0,
        decode_responses=True,
    )

    # 所有操作都是 await
    await r.set("async:hello", "world")
    value = await r.get("async:hello")
    print(value)  # 'world'

    # 用完关闭
    await r.aclose()

asyncio.run(main())
```

### 异步连接池

```python
import redis.asyncio as aioredis

# 异步连接池
pool = aioredis.ConnectionPool(
    host="127.0.0.1",
    port=6379,
    db=0,
    max_connections=30,
    decode_responses=True,
)

r = aioredis.Redis(connection_pool=pool)

# 在 FastAPI 中使用（预告，第八章详解）
# app.state.redis = r
```

### 同步 vs 异步：怎么选？

| | 同步 (redis.Redis) | 异步 (redis.asyncio) |
|---|---|---|
| **API** | `r.get("key")` | `await r.get("key")` |
| **适合** | 脚本、Flask、Django | FastAPI、Sanic |
| **性能** | 单线程阻塞 | 协程并发，吞吐更高 |
| **心智负担** | 低 | 需理解 async/await |

> 📌 **选择建议**：如果你的项目用 FastAPI，**一律用异步**。如果用 Flask/Django 或写脚本，同步就够了。

## 本章小结

| 知识点 | 要点 |
|--------|------|
| 安装 | `pip install redis`，服务端推荐 Docker 部署 |
| 连接 | `redis.Redis(...)` 或 `redis.from_url(...)` |
| decode_responses | 设为 `True` 返回 str，否则返回 bytes |
| 连接池 | 默认自动创建；显式创建可控制 `max_connections` |
| 异步 | `redis.asyncio` 模块，FastAPI 场景必选 |
| URL 连接 | `redis://[:password@]host:port/db`，适合环境变量 |
