> 本章目标：掌握固定窗口、滑动窗口、令牌桶三种限流算法的 Redis 实现，并封装成 FastAPI 中间件。

## 7.1 三种限流算法的 Redis 实现

### 为什么需要限流？

没有限流的 API 就像不设红绿灯的十字路口——高峰期必出事故：

```
无限流：
  用户 A: 1 秒内请求 1000 次 → 服务器过载 → 所有用户都受影响

有限流：
  用户 A: 1 秒内请求 1000 次 → 超过 100 次/秒的请求被拒绝
  用户 B: 正常使用不受影响
```

### 算法一：固定窗口计数器

最简单的限流——**在固定时间窗口内计数，超过阈值就拒绝**。

```
固定窗口（每分钟 100 次）：

  00:00 ─────── 00:59 │ 01:00 ─────── 01:59
       窗口 1           │      窗口 2
    计数: 0→1→2→...→100 │    计数重置: 0→1→2...
    超过100次 → 拒绝     │
```

```python
import redis
import time

r = redis.Redis(host="127.0.0.1", port=6379, db=0, decode_responses=True)


def fixed_window_limit(key: str, max_requests: int, window_seconds: int) -> bool:
    """
    固定窗口限流

    Args:
        key: 限流标识（如 "ratelimit:user:1001"）
        max_requests: 窗口内最大请求数
        window_seconds: 窗口大小（秒）

    Returns:
        True = 允许通过，False = 被限流
    """
    # 当前窗口的 key（按时间窗口分隔）
    window = int(time.time() // window_seconds)
    rate_key = f"{key}:{window}"

    current = r.incr(rate_key)

    if current == 1:
        # 首次请求，设置过期时间
        r.expire(rate_key, window_seconds + 1)

    return current <= max_requests


# 测试
for i in range(12):
    allowed = fixed_window_limit("ratelimit:api:login", max_requests=10, window_seconds=60)
    print(f"请求 {i+1}: {'✅ 通过' if allowed else '❌ 限流'}")
```

**固定窗口的问题：边界突刺**

```
窗口 1 的最后 1 秒：100 次请求
窗口 2 的最前 1 秒：100 次请求
→ 2 秒内实际承受了 200 次请求！超过预期的 2 倍
```

### 算法二：滑动窗口

滑动窗口解决了边界突刺问题——**窗口随时间滑动，而不是固定切割**。

用 Sorted Set 实现：每个请求记录时间戳作为 score，清理窗口外的旧记录，检查窗口内的总数。

```python
def sliding_window_limit(key: str, max_requests: int, window_seconds: int) -> bool:
    """
    滑动窗口限流（Sorted Set 实现）
    """
    now = time.time()
    window_start = now - window_seconds

    pipe = r.pipeline()

    # ① 清理窗口之外的旧记录
    pipe.zremrangebyscore(key, 0, window_start)

    # ② 统计当前窗口内的请求数
    pipe.zcard(key)

    # ③ 添加当前请求（用时间戳+随机数避免重复）
    member = f"{now}:{id(object())}"
    pipe.zadd(key, {member: now})

    # ④ 设置整个 key 的过期时间（兜底清理）
    pipe.expire(key, window_seconds + 1)

    results = pipe.execute()
    current_count = results[1]  # zcard 的结果

    if current_count >= max_requests:
        # 超限，删除刚刚添加的记录
        r.zrem(key, member)
        return False

    return True


# 测试：每 10 秒最多 5 次
for i in range(8):
    allowed = sliding_window_limit("ratelimit:slide:user:1001", max_requests=5, window_seconds=10)
    print(f"请求 {i+1}: {'✅ 通过' if allowed else '❌ 限流'}")
    time.sleep(0.1)
```

### 算法三：令牌桶

令牌桶允许一定程度的**突发流量**——桶里有令牌就放行，没有就拒绝。令牌按固定速率补充。

```
令牌桶（容量 10，速率 2个/秒）：

  ┌──────────────────┐
  │ 🪙🪙🪙🪙🪙🪙🪙   │  ← 桶（当前 7 个令牌）
  │ 容量上限: 10      │
  └────────┬─────────┘
           │
    每秒补充 2 个 🪙
           │
  请求来了 → 取走 1 个 🪙 → ✅ 通过
  桶空了   → 没有 🪙     → ❌ 拒绝
```

用 Lua 脚本实现（原子操作）：

```python
_TOKEN_BUCKET_SCRIPT = r.register_script("""
local key = KEYS[1]
local capacity = tonumber(ARGV[1])      -- 桶容量
local rate = tonumber(ARGV[2])          -- 每秒补充速率
local now = tonumber(ARGV[3])           -- 当前时间戳
local requested = tonumber(ARGV[4])     -- 请求的令牌数

-- 获取桶的当前状态
local bucket = redis.call('HMGET', key, 'tokens', 'last_refill')
local tokens = tonumber(bucket[1]) or capacity  -- 默认满桶
local last_refill = tonumber(bucket[2]) or now

-- 计算需要补充的令牌数
local elapsed = now - last_refill
local refill = elapsed * rate
tokens = math.min(capacity, tokens + refill)

-- 尝试消费令牌
if tokens >= requested then
    tokens = tokens - requested
    redis.call('HMSET', key, 'tokens', tokens, 'last_refill', now)
    redis.call('EXPIRE', key, capacity / rate * 2)  -- 兜底过期
    return 1  -- 允许
else
    redis.call('HMSET', key, 'tokens', tokens, 'last_refill', now)
    redis.call('EXPIRE', key, capacity / rate * 2)
    return 0  -- 拒绝
end
""")


def token_bucket_limit(
    key: str,
    capacity: int = 10,
    rate: float = 2.0,
    requested: int = 1,
) -> bool:
    """
    令牌桶限流

    Args:
        key: 限流标识
        capacity: 桶容量（最大突发量）
        rate: 每秒补充的令牌数
        requested: 本次请求消耗的令牌数

    Returns:
        True = 允许，False = 拒绝
    """
    now = time.time()
    result = _TOKEN_BUCKET_SCRIPT(
        keys=[f"ratelimit:bucket:{key}"],
        args=[capacity, rate, now, requested],
    )
    return result == 1


# 测试：容量 5，每秒补 2 个
print("--- 突发测试 ---")
for i in range(7):
    allowed = token_bucket_limit("api:search", capacity=5, rate=2)
    print(f"请求 {i+1}: {'✅' if allowed else '❌'}")

print("\n--- 等 2 秒后（补充 4 个令牌）---")
time.sleep(2)

for i in range(5):
    allowed = token_bucket_limit("api:search", capacity=5, rate=2)
    print(f"请求 {i+1}: {'✅' if allowed else '❌'}")
```

### 三种算法对比

| | 固定窗口 | 滑动窗口 | 令牌桶 |
|---|---|---|---|
| **实现复杂度** | ⭐ 最简单 | ⭐⭐ 中等 | ⭐⭐⭐ 较复杂 |
| **内存消耗** | 低（一个计数器） | 中（每个请求一条记录） | 低（两个字段） |
| **边界突刺** | ❌ 有 | ✅ 无 | ✅ 无 |
| **允许突发** | ❌ | ❌ | ✅（桶容量） |
| **精确度** | 低 | 高 | 高 |
| **适合** | 简单场景 | 精确限流 | API 网关 |

> 📌 **选择建议**：
> - 内部 API、不敏感 → **固定窗口**（简单够用）
> - 面向用户、需精确 → **滑动窗口**
> - API 网关、允许突发 → **令牌桶**

## 7.2 FastAPI 限流中间件实战

### 目标效果

```python
@app.get("/api/search")
@rate_limit(max_requests=10, window=60)  # 每分钟 10 次
async def search(q: str):
    return {"results": [...]}
```

### 实现限流中间件

```python
from fastapi import FastAPI, Request, HTTPException
from fastapi.responses import JSONResponse
import redis.asyncio as aioredis
import functools

app = FastAPI()

# 异步 Redis 连接
redis_pool = aioredis.ConnectionPool(
    host="127.0.0.1", port=6379, db=0,
    max_connections=30, decode_responses=True,
)
redis_client = aioredis.Redis(connection_pool=redis_pool)


# ── 方式一：全局中间件 ──

@app.middleware("http")
async def rate_limit_middleware(request: Request, call_next):
    """全局限流中间件：每个 IP 每分钟 60 次"""
    client_ip = request.client.host
    key = f"ratelimit:global:{client_ip}"
    window = 60
    max_requests = 60

    # 固定窗口限流
    window_id = int(time.time() // window)
    rate_key = f"{key}:{window_id}"

    current = await redis_client.incr(rate_key)
    if current == 1:
        await redis_client.expire(rate_key, window + 1)

    # 设置限流响应头
    response = None
    if current > max_requests:
        response = JSONResponse(
            status_code=429,
            content={"detail": "请求过于频繁，请稍后再试"},
        )
    else:
        response = await call_next(request)

    # 添加限流信息到响应头
    response.headers["X-RateLimit-Limit"] = str(max_requests)
    response.headers["X-RateLimit-Remaining"] = str(max(0, max_requests - current))
    response.headers["X-RateLimit-Reset"] = str((window_id + 1) * window)

    return response


# ── 方式二：路由级装饰器 ──

def rate_limit(max_requests: int = 10, window: int = 60, key_func=None):
    """
    路由级限流装饰器

    Args:
        max_requests: 窗口内最大请求数
        window: 窗口大小（秒）
        key_func: 自定义限流 key 的生成函数
    """
    def decorator(func):
        @functools.wraps(func)
        async def wrapper(*args, request: Request, **kwargs):
            # 生成限流 key
            if key_func:
                identifier = key_func(request)
            else:
                identifier = request.client.host

            rate_key = f"ratelimit:{func.__name__}:{identifier}:{int(time.time() // window)}"

            current = await redis_client.incr(rate_key)
            if current == 1:
                await redis_client.expire(rate_key, window + 1)

            if current > max_requests:
                raise HTTPException(
                    status_code=429,
                    detail={
                        "error": "Rate limit exceeded",
                        "limit": max_requests,
                        "window": f"{window}s",
                        "retry_after": window - int(time.time() % window),
                    },
                )

            return await func(*args, request=request, **kwargs)
        return wrapper
    return decorator


# ── 使用示例 ──

@app.get("/api/search")
@rate_limit(max_requests=10, window=60)
async def search(request: Request, q: str = ""):
    """搜索接口：每 IP 每分钟 10 次"""
    return {"query": q, "results": ["result1", "result2"]}


@app.post("/api/login")
@rate_limit(max_requests=5, window=300)  # 5 分钟 5 次
async def login(request: Request):
    """登录接口：严格限流防暴力破解"""
    return {"status": "ok"}


# 按用户 ID 限流（而非 IP）
def user_key(request: Request) -> str:
    return request.headers.get("X-User-ID", request.client.host)

@app.get("/api/export")
@rate_limit(max_requests=3, window=3600, key_func=user_key)  # 每小时 3 次
async def export_data(request: Request):
    """导出接口：按用户 ID 限流"""
    return {"status": "exporting"}
```

### 限流响应规范

遵循 HTTP 标准，返回 **429 Too Many Requests** 并附带有用的响应头：

```
HTTP/1.1 429 Too Many Requests
X-RateLimit-Limit: 10          ← 窗口内最大请求数
X-RateLimit-Remaining: 0       ← 剩余可用次数
X-RateLimit-Reset: 1709001200  ← 窗口重置时间（Unix 时间戳）
Retry-After: 45                ← 建议等待秒数

{
    "error": "Rate limit exceeded",
    "limit": 10,
    "window": "60s",
    "retry_after": 45
}
```

### 多维度限流

实际项目通常需要多个维度的限流叠加：

```python
@app.get("/api/ai/chat")
async def ai_chat(request: Request, message: str = ""):
    """AI 聊天接口：三层限流"""
    ip = request.client.host
    user_id = request.headers.get("X-User-ID", ip)

    checks = [
        # (key, 限制, 窗口, 描述)
        (f"ratelimit:ai:sec:{ip}", 2, 1, "每秒 2 次"),
        (f"ratelimit:ai:min:{user_id}", 20, 60, "每分钟 20 次"),
        (f"ratelimit:ai:day:{user_id}", 500, 86400, "每天 500 次"),
    ]

    for key, limit, window, desc in checks:
        rate_key = f"{key}:{int(time.time() // window)}"
        current = await redis_client.incr(rate_key)
        if current == 1:
            await redis_client.expire(rate_key, window + 1)
        if current > limit:
            raise HTTPException(status_code=429, detail=f"超出限制: {desc}")

    return {"reply": "AI 回复..."}
```

## 本章小结

| 知识点 | 要点 |
|--------|------|
| 固定窗口 | INCR + EXPIRE，简单但有边界突刺 |
| 滑动窗口 | Sorted Set，精确但内存消耗高 |
| 令牌桶 | Lua 脚本实现，允许突发流量 |
| 全局中间件 | `@app.middleware("http")`，所有路由生效 |
| 路由装饰器 | `@rate_limit()`，按接口设置不同限制 |
| 多维度限流 | 秒级 + 分钟级 + 天级叠加 |
| 响应规范 | 429 + X-RateLimit-* 响应头 |
